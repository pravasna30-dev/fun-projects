# Companion Document Template

One markdown file per hymn, named `<hymn-name>-companion.md`. Structure:

```markdown
# <Hymn Name> — Companion Guide for Recording & YouTube Chapters

Source text: `<source-file>` (note what it contains: transliteration only?
translation included? which script/convention?). English meanings below are
cross-checked against published translations (list the sites actually used)
— see notes at the end for anything genuinely disputed among scholars.

**Total chapters: N** — <list what they are, e.g. "Invocation, Verses 1–30,
the closing narrative verse, the Dhyana verse, and the final salutation
verse">. (Note here if a colophon/end-note line is deliberately excluded
from the chantable chapter list, and why.)

Each entry below = one YouTube chapter = one audio clip when recording.

<transliteration lines exactly as in the source>

**Meaning**
<verified English meaning, in your own words but faithful to the
cross-checked sources>

# YouTube Chapter / Description Template

<Note the audio source here once known: file name, duration, license status
— e.g. "used under Creative Commons Attribution, credit line needed" or
"self-recorded, no rights issues.">

**Label timestamps clearly as estimated vs. exact.** If timestamps are
exact (built from separately-recorded clips + fixed silence gaps), say so.
If they're a proportional/silencedetect-based estimate, say so explicitly
and flag it needs checking by ear before publishing.

YouTube chapter rules to keep in mind:
- First chapter **must** start at `0:00`.
- At least 3 chapters, each **at least 10 seconds** long.
- Timestamps must be in ascending order, format `M:SS` or `H:MM:SS`.

\`\`\`
0:00 <Chapter 1 label>
M:SS <Chapter 2 label>
...
\`\`\`

Flag with ⚠ any chapter that doesn't clear the 10-second minimum at the
recording's actual pace, and suggest the fix (pair verses, or add a
deliberate pause per stanza).

---

# Notes / open questions

1. Any verse where scholarly interpretation is genuinely disputed — name it,
   give the most commonly published reading, and say it's disputed rather
   than presenting it as settled.
2. Audio source status — is it still open, resolved, self-recording planned,
   etc.
```
